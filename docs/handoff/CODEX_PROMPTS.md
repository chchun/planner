# Codex CLI 작업 요청 프롬프트 (복사해서 사용)

리포 루트에서 `codex` 실행 후, 아래 블록을 순서대로 붙여넣으세요.
(전제: `AGENTS.md`는 리포 루트, `SPEC.md`·`TOKENS.md`·`prototype.html`은 `docs/`에 배치됨)

---

## STEP 0 — 시작 / 설계 승인

```
이 리포의 AGENTS.md 를 지침으로 삼아 작업한다.
먼저 docs/SPEC.md, docs/TOKENS.md 를 읽고 docs/prototype.html 을 열어 화면 동작을 파악해줘.
그다음 코드는 아직 작성하지 말고, 아래 3가지를 제안하고 내 승인을 기다려:
1) 프로젝트 폴더 구조 (컴포넌트 파일 분해 포함)
2) tailwind.config 토큰 매핑안 (TOKENS.md 기반)
3) 공유 상태 Context 설계 (과목/타이머/메모/할일 데이터 타입)
스캐폴딩(vite + react + ts + tailwind)까지는 진행해도 좋되, 화면 구현은 승인 후 시작해.
```

## STEP 1 — 대시보드

```
대시보드 화면(SCR-01)을 SPEC.md 2장 기준으로 구현해줘.
- 이월 숙제 위젯(항목 있을 때만), 마감 임박 타임라인(D-Day 배지 색상 규칙), 오늘의 할 일
- 체크 토글 → line-through + opacity, 서브태스크 아코디언
- mock 데이터는 prototype.html 의 state 값을 그대로 사용
시각은 docs/prototype.html 의 대시보드 탭을 기준으로 맞추고, 완료되면 멈춰서 확인받아.
```

## STEP 2 — 플래너

```
플래너 화면(SCR-04)을 SPEC.md 5장 기준으로 구현해줘.
- 06~24시 세로 시간표에 과목 색상 블록 배치(시작/종료 → top/height)
- 과목별 학습 계획 체크리스트(목표시간·메모·완료), 목표 대비 실제 요약 카드
- 과목 색상은 Context에서 공유(타이머와 동일 소스)
시각 기준은 prototype.html 플래너 탭. 완료 후 확인받아.
```

## STEP 3 — 타이머

```
타이머 화면(SCR-05)을 SPEC.md 6장 기준으로 구현해줘.
- 과목별 스톱워치: setInterval 초 단위, 동시 1과목만(다른 과목 시작 시 이전 정지), 언마운트 시 clear
- 상단 오늘 총 공부시간 + 진행 과목 표시, 진행 버튼 펄스
- 주간 공부량 막대 그래프(오늘 강조)
- fmtHMS / fmtHM 유틸 사용. 완료 후 확인받아.
```

## STEP 4 — 캘린더

```
캘린더 화면(SCR-02)을 SPEC.md 3장 기준으로 구현해줘.
- 월간/주간 뷰 전환
- 월간: iPad 우측 상세 패널 / iPhone 바텀시트, 일정=파란점·마감=주황점, 필터 칩
- 주간: 상세 패널 없이 캘린더를 풀스크린으로 확장하고 각 날짜 칸에 세부 일정 직접 표시
시각 기준은 prototype.html 캘린더 탭. 완료 후 확인받아.
```

## STEP 5 — 메모 보드

```
메모 보드 화면(SCR-06)을 SPEC.md 7장 기준으로 구현해줘.
- 작성: 텍스트 + 포스트잇 색상 + 폴더 선택 + 이미지 업로드(File API → dataURL 미리보기)
- 카드뷰/리스트뷰 전환, 폴더 칩 필터, 메모 삭제
- 이미지는 있을 때만 <img> 렌더
시각 기준은 prototype.html 메모 탭. 완료 후 확인받아.
```

## STEP 6 — 등록 모달

```
숙제 등록 모달(SCR-03)을 SPEC.md 4장 기준으로 구현해줘.
- 공용 폼 컴포넌트를 iPad 다이얼로그 / iPhone 바텀시트로 감싸기
- 필드: 제목, 태그 칩(가로 스크롤·단일선택), 납기일시(datetime-local, 필수*), 다중 알림 토글, 하위 항목 동적 추가/삭제
- 검증: 납기일시 미입력 저장 시 인라인 에러 + 테두리 red + 제출 차단, 입력되면 해제
완료 후 확인받아.
```

## STEP 7 — 반응형/마무리

```
전체 반응형을 정리해줘.
- iPad(가로): 사이드바 내비 5탭, 타이머 실행 중 표시
- iPhone(세로): 하단 탭바 + 중앙 FAB(등록)
- 모든 화면 prototype.html 과 시각 대조, 어긋나는 색·간격·라운드·그림자는 TOKENS.md 기준으로 수정
- 접근성(히트영역 44px, 포커스) 점검하고, 최종 빌드가 통과하는지 확인해줘.
```
