# 학습 관리 웹앱 — 개발 명세서 (SPEC)

> **핸드오프 목적**: 이 문서 + `prototype.html`(실행 가능한 목업) + `TOKENS.md`(디자인 토큰)를 기반으로
> React + Tailwind CSS + TypeScript 프로덕션 코드를 구현한다.
> **시각/인터랙션의 최종 기준은 언제나 `prototype.html`이다.** 애매하면 목업을 그대로 따른다.

---

## 0. 기술 스택 / 전제

- **프레임워크**: React 18 + TypeScript
- **스타일**: Tailwind CSS (토큰은 `TOKENS.md` → `tailwind.config` 확장으로 매핑)
- **폰트**: Pretendard (웹폰트 CDN 또는 로컬 번들)
- **상태관리**: 화면 로컬은 `useState`, 과목/타이머/메모 등 화면 간 공유 데이터는 Context(또는 Zustand) 1개
- **타깃 디바이스**: ① **iPad(가로, 학생 주 단말)** ② PC 웹(부모 조회·등록) ③ 안드로이드 폰(세로, 부모 보조).
  - iPad·PC: 좌측 고정 사이드바 내비게이션 (PC는 콘텐츠 최대폭 제한)
  - 모바일: 하단 탭바(5탭) + FAB(등록)
  - **주의**: 이 문서와 prototype.html의 "iPhone"은 작성 당시 표기로, **"모바일 세로 레이아웃(안드로이드 Chrome 기준)"으로 읽는다.** iPhone은 타깃이 아님
- **데이터**: Phase 1은 mock 데이터로 동작. API 연동 지점만 주석/함수로 분리.

## 1. 전역 구조 (App Shell)

### 네비게이션 (5개 탭)
`dashboard` · `planner` · `timer` · `calendar` · `memo`

- **iPad**: 사이드바 항목 = 로고 / 대시보드 / 플래너 / 타이머 / 캘린더 / 메모 / (하단)「새 숙제 등록」버튼 / 사용자 프로필
  - 타이머 실행 중이면 사이드바 "타이머" 항목에 빨간 점(펄스) 표시
- **iPhone**: 하단 탭바 = 홈 / 플래너 / (중앙 FAB) / 타이머 / 캘린더 — 메모는 별도 진입(탭바 확장 또는 홈 바로가기). 목업 기준으로 배치.
- 헤더: 좌측 날짜 + 화면 타이틀, 우측 프로필 아바타

### 공유 상태 (Context)
```ts
type Subject = { name: string; color: string; todaySec: number; weekSec: number };
// 과목: 수학(#4f46e5) 영어(#0ea5e9) 국어(#f97316) 과학(#10b981) 사회(#a855f7)
// 과목 color는 플래너 시간표 블록 / 계획 리스트 / 타이머 / 주간 그래프에서 공유한다.
```

## 2. SCR-01 대시보드 (`<Dashboard />`)

수직 스택(iPhone) / 상단 전폭 + 2단 그리드(iPad).

- **이월 숙제 위젯 `<OverdueWidget />`**
  - `is_overdue === true` 항목이 있을 때만 렌더.
  - 연한 붉은 배경 `bg-red-50` + 좌측 굵은 테두리 `border-l-4 border-red-500`, 우측 경고 아이콘, 개수 배지.
- **마감 임박 타임라인 `<UrgentTimeline />`**
  - 만료 임박 순 정렬 카드 리스트. 각 카드: 좌측 색상 바 + 제목 + 출처·남은시간 + 우측 D-Day 배지.
  - 배지 색상 규칙: D-DAY/시간단위=red, D-1~D-2=amber, D-3 이상=emerald(green).
- **오늘의 할 일 `<DailyTodoList />`**
  - 우선순위 칩(상=red / 중=amber / 하=emerald), 출처 카테고리 태그(`text-xs bg-slate-100`).
  - 체크박스 `onToggle` → 해당 아이템 `line-through` + `opacity-50`.
  - 서브태스크 존재 시 화살표 버튼 → `isOpen` 토글로 슬라이드다운 아코디언. 서브태스크도 개별 체크 가능.

## 3. SCR-02 캘린더 (`<CalendarView />`)

- **뷰 전환 토글**: [월간] / [주간]
- **월간 뷰**: `grid grid-cols-7` 월간 달력.
  - iPad는 좌측 큰 달력 + **우측 상세 패널**(선택 날짜 일정 시간순). 날짜 미선택 시 빈 상태 안내.
  - iPhone은 달력 + 날짜 클릭 시 **바텀 시트**(슬라이드업)로 상세.
  - 필터 칩 `<ScheduleFilter />`: [학교/학원 일정](파란 점) / [숙제 마감일](주황 점) 토글.
  - Color-coding: 학원/학교 일정 = 파란 점, 숙제 마감 = 주황 점.
  - 오늘 강조, 일요일=red / 토요일=blue 텍스트.
- **주간 뷰**: 상세 패널/바텀시트 **불필요**. 캘린더 카드를 **전체 폭(풀스크린)**으로 확장하고, 각 날짜 칸 안에 해당일 일정·마감을 **세부 내용(제목·시간 칩)까지** 직접 표시한다.

## 4. SCR-03 숙제/태스크 등록 모달 (`<RegisterModal />`)

- **레이어**: iPad = 중앙 다이얼로그(`fixed inset-0 bg-black/50 z-50`), iPhone = 바텀 시트.
- **공용 폼 컴포넌트**로 분리(두 디바이스가 동일 폼 재사용).
- 필드:
  - 제목 input
  - 태그 선택: 가로 스크롤 칩 셀렉터(단일 선택)
  - **납기 일시 `datetime-local` (필수 `*`)**
  - 다중 알림 토글 셋: [전날 21:00] / [당일 아침] / [마감 2시간 전] 스위치
  - 서브태스크: 「하위 항목 추가」 → input 동적 추가/삭제(배열 상태)
- **유효성 검사**: 납기 일시 미입력 상태로 저장 클릭 시 → 인라인 에러(`text-red-500 text-sm`) + 필드 테두리 red + 제출 차단. 입력되면 에러 해제.

## 5. SCR-04 플래너 뷰 (`<PlannerView />`)  *(스터디 플래너)*

- **오늘의 시간표**: 06:00~24:00 세로 타임라인. 계획된 공부 블록을 과목 색상으로 절대 위치 배치(시작~종료 시각 = top/height). 블록에 과목명 + 시간 라벨.
- **과목별 학습 계획 리스트**: 각 행 = 체크박스 + 과목 색상 바 + "과목 · 메모" + 목표시간. 체크 시 완료 처리(취소선/흐림). 진행도 `n/총 완료`.
- **요약 카드**: 오늘 목표 공부시간 vs 실제 공부시간(타이머 누적과 연동).

---

## 6. SCR-05 타이머 뷰 (`<TimerView />`)  *(열품타 스타일)*

- **총 공부시간 대형 타이머**: 오늘 전체 누적 `H:MM:SS`, 현재 진행 과목 라벨/색상.
- **과목별 타이머 리스트**: 각 행 = 과목 색상 + 이름 + 이번 주 누적 + 오늘 누적(H:MM:SS) + 재생/정지 버튼.
  - **동시 1과목만 진행.** 다른 과목 시작 시 이전 과목 자동 정지.
  - 진행 중 버튼은 과목 색상 + 펄스(ring) 애니메이션.
  - 1초 간격으로 `todaySec` / `weekSec` 증가(`setInterval`, 언마운트 시 clear).
- **주간 공부량 막대 그래프**: 월~일 7개 막대, 오늘 강조. 주 합계 표시.

### 시간 포맷 유틸
- `fmtHMS(sec)` → `H:MM:SS` (타이머 표시)
- `fmtHM(sec)` → `N시간 M분` (누적 요약)
- `fmtMin(min)` → `N시간 M분` (계획 목표)

## 7. SCR-06 메모 보드 (`<MemoBoard />`)

- **작성**: 텍스트 입력 + 색상(포스트잇) 선택 + 폴더 선택 + **이미지(스크린샷) 업로드**(File API, `FileReader` → dataURL 미리보기).
- **뷰 전환**: [카드뷰](포스트잇 그리드) / [리스트뷰](행). 메모가 많아지면 리스트뷰로 대량 열람.
- **폴더**: 메모를 폴더로 분류. 상단 폴더 칩으로 필터. (예: 전체 / 수학 / 영어 / 오답노트 …)
- **이미지 메모**: 카드에는 이미지 표시, 리스트에는 썸네일. **이미지가 있을 때만** `<img>` 렌더(없으면 미렌더).
- **삭제**: 각 메모 "확인 완료 → 삭제" 액션. 카드/리스트 모두 삭제 버튼.
- 빈 상태 안내 문구 제공.

## 8. Mock 데이터 규격 (초기 화면이 풍성하도록 미리 삽입)

- **과목**: 수학/영어/국어/과학/사회 (색상은 위 규격)
- **오늘의 할 일**: 수학 학원 숙제(미적분 p.45-50, 서브태스크 3), 통합과학 수행평가(서브 2), 영어 단어 100개(완료), 국어 비문학 2지문
- **이월 숙제**: 화학 실험 보고서(어제 마감), 수학 문제집 30문항(2일 전)
- **마감 임박**: 통합과학 수행평가(D-DAY), 영어 단어 시험(3시간), 국어 독서록(D-1), 물리 노트(D-3)
- **캘린더 이벤트**: 날짜별 sched/hw 혼합(목업의 `events` 참조)
- **플래너 계획/시간표**: 국어·수학·영어·과학 블록(목업의 `timetable`/`plan` 참조)
- **메모**: 텍스트/이미지 혼합 예시 몇 건, 폴더 2~3개

각 mock의 실제 값·초기 상태는 `prototype.html`의 `Component` 클래스 `state`를 그대로 참조할 것.
