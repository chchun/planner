# BACKLOG — 기능 백로그

> 새 기능 아이디어를 여기에 누적하고, 착수할 항목을 골라 spec 번호를 부여한다 (다음: **007**).
> 프로세스: 백로그 선정 → spec → plan → tasks → 구현 → 검증 → 마감 (CLAUDE.md SDD 워크플로 참조)
> 상세 프로세스·착수 프롬프트: [Confluence — backlog & process](https://s-cloud.atlassian.net/wiki/spaces/~7120207d759bee866546f299b325291f6cf829/pages/1133445121/backlog+process)

## 사용 규칙
- 아이디어가 생기면 **등록일과 함께** 해당 분류 표에 추가한다
- 착수가 결정되면 상태를 `진행(spec 00N)`으로 바꾸고 spec 작성을 시작한다
- 구현·배포까지 끝나면 행을 맨 아래 **완료 이력**으로 옮긴다 (삭제하지 않는다)
- 규모 기준: 소(1~2일) / 중(3~5일) / 대(1주+)
