# AGENTS.md — 학습 관리 웹앱 (Phase 1)

> 이 파일은 Codex CLI가 세션마다 자동으로 읽는 프로젝트 규칙이다. 항상 준수한다.
> **첫 빌드는 Claude Code가 SDD 방식으로 진행한다. 규칙이 충돌하면 루트의 `CLAUDE.md`와 `specs/`가 우선한다.**

## 프로젝트 목표
고등학생 학습 관리 웹앱 Phase 1을 **React + TypeScript + Tailwind CSS**로 구현한다.
`docs/prototype.html`은 실행 가능한 목업으로, **시각·레이아웃·인터랙션의 최종 기준**이다.
애매하면 목업을 그대로 따른다.

## 반드시 참고할 문서
- `docs/SPEC.md` — 화면별 컴포넌트 구조, 상태, 인터랙션, mock 데이터 규격
- `docs/TOKENS.md` — 컬러/폰트/간격/모션 토큰 → `tailwind.config`에 매핑
- `docs/prototype.html` — 브라우저로 열어 실제 동작 확인 (읽어서 마크업·스타일 참조)

## 기술 규칙
- React 18 + TypeScript, Tailwind CSS, 폰트는 Pretendard
- 상태: 화면 로컬은 `useState`, 과목/타이머/메모 등 공유 데이터는 Context 1개(또는 Zustand)
- 타깃: **① iPad(가로, 학생 주 단말) ② PC 웹(부모 조회·등록) ③ 안드로이드 폰(세로, 부모 보조)**. iPhone은 타깃 아님. iPad·PC는 사이드바, 모바일은 하단 탭바 + FAB
- 컴포넌트는 SPEC.md의 분해안을 따라 파일 단위로 분리
- Phase 1은 mock 데이터로 동작하되, API 연동 지점은 별도 함수/주석으로 분리
- 접근성: 히트영역 44px 이상, 시맨틱 태그, 키보드 포커스

## 핵심 기능 체크리스트
- 타이머: 시간 누적은 시작 시각(`startedAt`) 타임스탬프 기반으로 계산(`setInterval`은 화면 갱신용으로만, 언마운트 시 clear). 동시 1과목. 과목 색상은 플래너/타이머/그래프 공유
- 등록 모달: 납기일시(`datetime-local`) 필수 검증 — 미입력 저장 시 인라인 에러 + 테두리 red + 제출 차단. 공용 폼을 iPad·PC 다이얼로그 / 모바일 바텀시트로 감쌀 것
- 캘린더: 월간(상세 패널/바텀시트) + 주간(풀스크린, 각 날짜 칸에 세부 일정 직접 표시)
- 메모 보드: 카드뷰/리스트뷰 전환, 폴더 필터, 이미지 업로드(File API dataURL), 삭제. 이미지는 있을 때만 렌더

## 작업 방식
- 한 번에 전체를 만들지 말고 화면 단위로 완성하고 멈춰서 확인받는다:
  **대시보드 → 플래너 → 타이머 → 캘린더 → 메모 보드 → 등록 모달 → 반응형 정리**
- 시작 전 프로젝트 구조 · `tailwind.config` · Context 설계를 먼저 제안한다
- 각 화면 완성 후 의미 있는 단위로 커밋한다
- 파일을 새로 만들거나 명령을 실행하기 전 사용자 승인 흐름을 존중한다
