# Spec 001 — Phase 1: 프론트엔드 UI (mock 데이터)

> **What만 정의한다.** 화면별 상세(컴포넌트 구조·스타일·mock 값)는 `docs/SPEC.md`와
> `docs/prototype.html`을 정본으로 참조하고, 여기서는 요구사항과 수용 기준만 관리한다.

## 범위
- 포함: 앱 셸(내비게이션), 대시보드, 플래너, 타이머, 캘린더, 메모 보드, 등록 모달, iPad/PC/모바일 반응형
- 제외: 로그인/인증, 서버 연동, 오프라인 저장, Google Calendar (→ `docs/ROADMAP.md` Phase 2~4)

## 사용자 / 단말 우선순위
- 사용자: 고등학생 1인(주 사용자) + 부모(엄마·아빠 — 조회 및 메모 등 데이터 등록)
- **1순위: iPad(가로)** — 학생 주 단말. 모든 요구사항의 기준 뷰포트이며, 구현·검증을 항상 iPad부터 한다
- **2순위: PC 웹(데스크톱)** — 부모가 브라우저로 조회·등록. 사이드바 레이아웃을 공유하되 넓은 화면(1280px+)에서 깨지지 않아야 함
- **3순위: 안드로이드 폰(세로, Chrome)** — 부모 보조 단말. 하단 탭바 + FAB 모바일 레이아웃
- **iPhone은 타깃이 아니다.** 문서·목업의 "iPhone 레이아웃"은 "모바일 세로 레이아웃"으로 읽는다
- 부모별 권한 구분(조회/등록 범위)은 Phase 2에서 다룬다 — Phase 1은 단일 사용자 mock

## R-01 앱 셸 / 내비게이션  *(docs/SPEC.md §1)*
5개 영역(dashboard · planner · timer · calendar · memo) 간 이동.

**수용 기준**
- [ ] iPad(가로)·PC: 좌측 고정 사이드바 — 로고/5개 메뉴/「새 숙제 등록」버튼/프로필. PC 넓은 화면에서는 콘텐츠 최대폭 제한으로 깨짐 방지
- [ ] 모바일(안드로이드 세로): 하단 탭바 + 중앙 FAB(등록). 메모는 목업 기준 배치
- [ ] 타이머 실행 중이면 사이드바 "타이머" 항목에 빨간 점(펄스) 표시
- [ ] 헤더: 좌측 날짜+타이틀, 우측 프로필 아바타
- [ ] 화면 전환 시 각 화면의 상태(입력 중 값 제외한 공유 상태)가 유지된다

## R-02 대시보드  *(docs/SPEC.md §2)*
오늘 해야 할 일과 마감 위험을 한눈에 보여준다.

**수용 기준**
- [ ] 이월 숙제 위젯: `is_overdue` 항목이 있을 때만 렌더, 경고 스타일 + 개수 배지
- [ ] 마감 임박 타임라인: 임박 순 정렬, D-Day 배지 색상 규칙(D-DAY/시간=red, D-1~2=amber, D-3+=green)
- [ ] 오늘의 할 일: 우선순위 칩, 체크 시 취소선+흐림, 서브태스크 아코디언 + 개별 체크

## R-03 플래너  *(docs/SPEC.md §5)*
오늘의 학습 계획을 시간표와 체크리스트로 관리한다.

**수용 기준**
- [ ] 06:00~24:00 세로 타임라인에 계획 블록을 과목 색상으로 절대 위치 배치(시작~종료 = top/height)
- [ ] 과목별 계획 리스트: 체크 시 완료 처리, 진행도 `n/총` 표시
- [ ] 요약 카드: 오늘 목표 시간 vs 실제 공부 시간(타이머 누적과 실시간 연동)

## R-04 타이머  *(docs/SPEC.md §6)*
과목별 공부 시간을 측정한다(열품타 스타일).

**수용 기준**
- [ ] 총 공부시간 대형 타이머 `H:MM:SS` + 현재 과목 라벨/색상
- [ ] 과목별 행: 주간/오늘 누적 + 재생·정지. **동시 1과목만** — 다른 과목 시작 시 이전 과목 자동 정지
- [ ] 진행 중 버튼: 과목 색상 + ring 펄스
- [ ] **탭을 백그라운드로 보냈다가 복귀해도 경과 시간이 정확하다** (타임스탬프 기반 계산 검증)
- [ ] 주간 막대 그래프: 월~일 7개, 오늘 강조, 주 합계

## R-05 캘린더  *(docs/SPEC.md §3)*
일정과 숙제 마감을 월간/주간으로 조회한다.

**수용 기준**
- [ ] 월간: 7열 그리드. iPad·PC는 우측 상세 패널, 모바일은 바텀 시트. 미선택 시 빈 상태 안내
- [ ] 필터 칩: 학교/학원 일정(파란 점) · 숙제 마감(주황 점) 토글
- [ ] 오늘 강조, 일요일 red / 토요일 blue
- [ ] 주간: 풀스크린 확장, 각 날짜 칸에 일정 세부(제목·시간 칩)를 직접 표시. 상세 패널 없음

## R-06 등록 모달  *(docs/SPEC.md §4)*
숙제/태스크를 등록한다.

**수용 기준**
- [ ] 공용 폼 1개를 iPad·PC 다이얼로그 / 모바일 바텀시트가 감싼다
- [ ] 필드: 제목, 태그 칩(단일), 납기 `datetime-local`(필수), 알림 토글 3종, 서브태스크 동적 추가/삭제
- [ ] 납기 미입력 저장 시: 인라인 에러 + 테두리 red + 제출 차단. 입력 시 에러 해제
- [ ] 저장 시 할 일 목록(대시보드)에 즉시 반영

## R-07 메모 보드  *(docs/SPEC.md §7)*
포스트잇형 메모를 작성·분류·열람한다.

**수용 기준**
- [ ] 작성: 텍스트 + 색상 선택 + 폴더 선택 + 이미지 업로드(FileReader → dataURL 미리보기)
- [ ] 카드뷰/리스트뷰 전환, 폴더 칩 필터
- [ ] 이미지가 있을 때만 `<img>` 렌더(카드=이미지, 리스트=썸네일)
- [ ] 삭제 액션, 빈 상태 안내

## R-08 반응형 / 공통
**수용 기준**
- [ ] **iPad 가로(1024×768)에서 모든 화면이 prototype.html과 일치** — 1순위 기준
- [ ] PC 데스크톱(1280px+, 1920px까지)에서 모든 화면 레이아웃 깨짐 없음 — 2순위 기준
- [ ] 안드로이드 폰 세로(~412px, Chrome)에서 모든 화면 레이아웃 깨짐 없음 — 3순위 기준
- [ ] 모든 인터랙티브 요소 히트영역 44px 이상
- [ ] mock 데이터는 docs/SPEC.md §8 규격 — 초기 화면이 풍성하게 보일 것
