# Spec 006 — 플래너 날짜별 관리

> 목표: 플래너에 날짜 네비게이션을 두고, **날짜마다 다른 계획**과 **요일별 시간표**를 본다.
> 로컬 개발 흐름(PGlite + `npm run server`/`npm run dev`)과 배포는 그대로 유지한다.

## 범위
- 포함: 날짜 네비게이션 UI(이전/다음/오늘/날짜선택), 날짜별 계획 CRUD, 요일별 시간표, 스키마 마이그레이션·시드
- 제외: 반복 일정 규칙, 계획 드래그 정렬, "특정 날짜만 예외 시간표"(요일 템플릿으로 충분), 오프라인 과거/미래 날짜 캐시

## 핵심 설계 결정
- **시간표 = 요일 템플릿**: 같은 요일은 같은 시간표(월요일 시간표는 매주 동일). `timetable_blocks.dow`로 구분
- **계획 = 날짜별**: 날짜마다 다른 계획. `plan_items.plan_date`로 구분
- 날짜 이동은 온라인 조회(과거/미래). 오프라인은 오늘 스냅샷만(Phase 3 범위 유지)

## 요구사항

### R-61 날짜 네비게이션
- [ ] 플래너 상단에 `◀ YYYY년 M월 D일 (요일) ▶` 헤더 + "오늘" 버튼
- [ ] ◀/▶ 로 하루 전/후 이동
- [ ] 날짜 클릭 시 **년/월/일 선택 팝업** → 선택하면 해당 날짜로 이동
- [ ] 기본값은 오늘. "오늘" 버튼으로 복귀

### R-62 날짜별 계획
- [ ] 계획(plan)은 선택 날짜에 종속 — 날짜 이동 시 그 날짜의 계획만 표시
- [ ] 계획 추가(과목·목표시간·메모) / 삭제 / 완료 체크
- [ ] 계획이 없는 날짜는 빈 상태 안내
- [ ] 오늘 목표 vs 실제(StudySummaryCard)의 목표시간은 선택 날짜 계획 기준

### R-63 요일별 시간표
- [ ] 시간표(timetable)는 요일(월~일) 템플릿 — 선택 날짜의 요일 블록을 표시
- [ ] 같은 요일은 같은 시간표(특정 날짜만 다른 시간표는 범위 밖)

### R-64 마이그레이션 · 시드
- [ ] `plan_items`에 `plan_date`, `timetable_blocks`에 `dow` 추가(기존 데이터 보존)
- [ ] 배포 반영: `npm run db:setup`(로컬) / `GET /api/admin/db-setup`(Neon) 재실행으로 스키마 적용
- [ ] 시드를 요일별 시간표 + 오늘 계획으로 갱신

## 비기능
- 데이터 접근은 repository 계층 유지(constitution §3). 날짜는 `YYYY-MM-DD`(KST 기준)로 주고받는다
- 로컬은 인터넷·계정 없이 동작(PGlite)
