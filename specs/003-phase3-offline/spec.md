# Spec 003 — Phase 3: 태블릿 오프라인 (좁은 범위)

> `docs/ROADMAP.md` Phase 3의 구체화. **범용 동기화 엔진을 만들지 않는다.**
> 오프라인 비율 ~20% 전제. PC 웹·부모 폰은 온라인 전용 유지.

## 범위
- 포함: ① 조회 캐시 ② 타이머 세션 오프라인 기록→업로드 ③ 할일·서브태스크·계획 **완료 체크** 큐
  ④ 동기화 상태 표시 ⑤ PWA(홈 화면 설치, 앱 셸 오프라인 로드, storage.persist)
- 제외(온라인 전용 유지): 할일·메모·일정 **신규 생성/삭제**, 로그인, 통계 재계산, 충돌 UI → 이후 검토

## 요구사항

### R-21 조회 캐시
- [ ] 온라인 로드 성공 시 전체 데이터(bootstrap)를 IndexedDB에 스냅샷 저장
- [ ] 오프라인(또는 서버 접속 실패) 시 캐시 스냅샷으로 앱이 정상 표시됨 — 대시보드/플래너/타이머/캘린더/메모 조회 가능
- [ ] 캐시가 없고 오프라인이면 로그인 화면에 "네트워크 연결 후 이용 가능" 안내
- [ ] 401(세션 만료)이면 캐시를 지우고 로그인 화면으로

### R-22 오프라인 쓰기 큐
- [ ] 오프라인 상태에서 다음 동작이 큐에 저장되고 UI에 즉시 반영: 할일 완료 체크, 서브태스크 체크, 계획 완료 체크, 타이머 세션 기록
- [ ] 같은 항목을 여러 번 토글하면 **마지막 상태만** 전송 (ROADMAP 합치기 규칙)
- [ ] 타이머 세션은 append-only — 모두 전송, 합치지 않음
- [ ] 네트워크 복구(online 이벤트/재시도) 시 큐를 순서대로 자동 전송 → 성공 후 서버 최신 데이터 재로드
- [ ] 충돌 정책: last-write-wins. 충돌 UI 없음
- [ ] 오프라인에서 신규 생성 UI(등록 모달 저장, 메모 추가)는 비활성 + 안내 문구

### R-23 동기화 상태 표시
- [ ] 상시 표시: 온라인/오프라인 상태, 전송 대기 건수, 마지막 동기화 시각 (iPad 사이드바 + 모바일 헤더)
- [ ] 수동 "지금 동기화" 동작 제공
- [ ] 앱 재시작 후에도 대기 큐가 유지됨 (IndexedDB)

### R-24 PWA
- [ ] manifest + service worker — 홈 화면 설치 후 오프라인에서도 앱 셸(HTML/JS/CSS)이 로드됨
- [ ] `navigator.storage.persist()` 호출 (iOS 저장소 삭제 방지, ROADMAP)
- [ ] API 요청은 SW가 캐시하지 않음 (데이터 캐시는 앱 레벨 IndexedDB만)

## 비기능
- 타이머 시간 계산은 기존 타임스탬프 방식 그대로 (constitution §4)
- 큐 항목 구조는 ROADMAP의 SyncQueueItem 인터페이스를 따름
- 오프라인 미지원 기능이 오류를 내지 않고 명확히 비활성으로 보일 것
