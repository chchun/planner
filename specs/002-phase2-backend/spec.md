# Spec 002 — Phase 2: 백엔드 + 온라인 전용 전 기능

> `docs/ROADMAP.md` Phase 2 항목의 구체화. Phase 1 UI는 유지하고 데이터 출처를 mock → 백엔드 API로 교체한다.

## 범위
- 포함: 백엔드 API 서버, PostgreSQL 스키마, 자체 로그인(학생/부모), Phase 1 전 기능의 서버 연동,
  타이머 세션 저장, 실제 날짜 기반 동작(고정 mock 날짜 제거), 마감(D-Day)·이월 숙제의 실데이터 파생
- 제외: 오프라인(IndexedDB), Google Calendar, 충돌 감지 활성화(version 컬럼은 예약만) → Phase 3~4

## 사용자 / 계정
- 계정 3개 시드: 학생 1(siyoon, 전시윤) + 부모 2(mom, dad). 공개 회원가입 없음(가족용)
- role: `student` | `parent`. **부모는 조회 + 등록 가능**(할일·메모 등). 타이머 세션 기록은 학생만

## 요구사항

### R-11 인증
- [ ] 아이디/비밀번호 로그인. 비밀번호는 해시 저장
- [ ] 세션 토큰: httpOnly 쿠키, 30일 유효 (태블릿 장기 세션 대비 — ROADMAP)
- [ ] 미로그인 시 로그인 화면만 표시. 로그아웃 가능
- [ ] API는 전부 인증 필수(로그인 제외). 401 시 클라이언트는 로그인 화면으로

### R-12 데이터 서버화
- [ ] 과목/할일(+서브태스크)/학습계획/시간표/캘린더 일정/메모가 PostgreSQL에 저장되고 API로 조회·변경
- [ ] 소프트 삭제(`deleted_at`), TIMESTAMPTZ, `version` 컬럼 예약 (constitution·ROADMAP)
- [ ] 등록 모달 저장 → 서버 저장 → 새로고침해도 유지
- [ ] 메모 작성·완료·삭제가 서버에 반영되어 다른 기기(PC↔iPad)에서 동일 조회

### R-13 실날짜 전환
- [ ] 고정 mock 날짜(2026-07-14) 제거 — 헤더/캘린더/주간이 실제 오늘 기준
- [ ] 할일에 납기(`due_at`) 저장. **마감 임박 타임라인 = 미완료+납기 있는 할일을 임박순 파생**
  (D-Day 배지 규칙은 SPEC §2 유지: 당일/시간단위=red, D-1~2=amber, D-3+=green)
- [ ] **이월 숙제 위젯 = 납기가 지난 미완료 할일** ("어제 마감", "N일 전 마감" 라벨)

### R-14 타이머 세션
- [ ] 타이머 정지(또는 과목 전환) 시 세션(startedAt~endedAt)을 서버에 append-only 저장
- [ ] 오늘/이번 주 누적, 주간 그래프(월~일)는 서버 세션 집계로 계산
- [ ] 클라이언트 표시 로직은 Phase 1의 타임스탬프 방식 유지 (constitution §4)

### R-15 역할 제어(최소)
- [ ] 타이머 세션 기록 API는 학생만 허용(서버 검증). 부모 UI에서는 시작 버튼 비활성
- [ ] 그 외 조회·등록은 학생/부모 모두 가능

## 비기능
- 브라우저에서 DB 직접 접속 금지 — API 경유 (ROADMAP)
- Phase 1 repository 계층 시그니처를 유지하며 구현체만 교체 (constitution §3)
- 서버 미기동/네트워크 오류 시 콘솔 에러와 함께 UI가 깨지지 않을 것(온라인 전용이므로 재시도 UI는 불요)
