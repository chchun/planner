# Spec 004 — Phase 4: Google Calendar 단방향 연동

> `docs/ROADMAP.md` Phase 4. 인증은 **서비스 계정**(키 발급·캘린더 공유 완료).
> 매핑(사용자 확정): 플래너 → **시윤학원** 캘린더로 push / **가족일정 + 시윤학원**을 앱 캘린더에 조회 표시.

## 범위
- 포함: ① 서비스 계정 인증 ② 숙제(납기 있는 할일) 생성 시 시윤학원 캘린더에 이벤트 push, 삭제 시 구글에서도 삭제
  ③ 가족일정·시윤학원 캘린더 일정을 앱 캘린더 화면에 읽어와 표시(조회 전용) ④ 실패 시 상태 기록 + 서버 재기동 시 재시도
- 제외: 양방향 동기화(구글→플래너 수정 반영), 구글 일정의 앱 내 편집, 주기적 백그라운드 재시도 데몬

## 요구사항

### R-31 서비스 계정 인증
- [ ] `GOOGLE_SERVICE_ACCOUNT_JSON`(base64 또는 원본 JSON)로 Calendar API 액세스 토큰 발급(JWT), 만료 전 캐시
- [ ] 키/환경변수 미설정 시 연동 기능만 비활성 — 앱의 다른 기능은 정상 동작
- [ ] 키는 git에 커밋되지 않는다 (.env, .gitignore)

### R-32 숙제 push (플래너 → 시윤학원)
- [ ] 납기(dueAt) 있는 할일 생성 시 시윤학원 캘린더에 이벤트 생성 — 제목 "[숙제] {title}", 시간 = 납기 30분 전~납기
- [ ] 생성된 `google_event_id`와 `google_sync_status`(pending/synced/failed)를 todos에 저장
- [ ] 할일 삭제 시 구글 이벤트도 삭제(실패해도 앱 삭제는 진행)
- [ ] push 실패가 할일 생성을 막지 않는다(생성은 성공, 상태만 failed)
- [ ] 서버 부팅 시 pending/failed 항목 재시도

### R-33 구글 캘린더 조회 표시
- [ ] 앱 캘린더 화면에 가족일정·시윤학원의 이번 달 일정이 함께 표시된다
- [ ] 가족일정 항목은 "[가족]" 접두어로 구분, 종류는 "일정"(파란 점) 취급
- [ ] 서버가 조회를 중계(클라이언트에 키 노출 금지), 5분 캐시로 구글 API 호출 최소화
- [ ] 구글 조회 실패 시 로컬 일정만으로 정상 동작(캘린더 화면이 깨지지 않음)
- [ ] 오프라인(Phase 3)에서는 스냅샷에 저장된 마지막 조회분이 그대로 보인다

## 비기능
- Google Calendar는 연동 대상일 뿐 — PostgreSQL이 항상 원본 (constitution·ROADMAP)
- 클라이언트 번들에 서비스 계정 정보가 포함되지 않을 것
