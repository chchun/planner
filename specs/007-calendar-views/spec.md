# Spec 007 — 캘린더 뷰 확장 (주간 타임그리드 + 일정 뷰)

## 배경
현재 캘린더는 월간/주간 두 가지 뷰가 있다. 주간 뷰는 날짜 칸에 일정을 목록으로만 나열해
시간대 감각이 없고, 하루 일정 흐름을 훑어보는 리스트 형태의 뷰가 없다.
구글 캘린더의 **주(시간축 그리드)** / **일정(아젠다 리스트)** 뷰를 기준으로 개선한다.

## 요구사항

### R-71. 뷰 전환에 "일정" 추가
- 캘린더 상단 뷰 토글을 `월 / 주 / 일정` 3개로 확장한다.
- 기존 필터 칩(학교/학원 일정, 숙제 마감일)은 세 뷰 모두에 적용된다.

### R-72. 주간 뷰 = 시간축 타임그리드 (구글 캘린더 방식)
- Y축에 시간 눈금(시 단위), X축에 이번 주 7일(일~토)을 놓는다.
- 각 일정은 **시작~종료 시각에 비례한 높이의 색 블록**으로 해당 요일 열에 배치한다.
  - 블록 안에 제목 + 시간 범위 표시
  - 종료 시각이 없는 일정(기존 데이터·숙제 마감)은 기본 1시간 블록으로 표시
  - 같은 시간대에 겹치는 일정은 열 안에서 좌우로 나눠 표시(겹침 분할)
- 시간축 범위는 그 주 일정을 모두 포함하되 최소 08:00~22:00을 보장한다.
- 오늘 열은 시각적으로 구분(요일 헤더 원형 강조)하고, 오늘 열에 **현재 시각 빨간 선**을 표시한다.
- 모바일(세로)에서는 같은 그리드를 가로 스크롤로 제공한다.

### R-73. 일정(아젠다) 뷰 (구글 캘린더 방식)
- 이번 달 일정이 있는 날짜만 위→아래로 나열한다. 날짜 행: 날짜 숫자(오늘은 파란 원)
  + `M월, 요일` 라벨, 그 옆에 그 날의 일정 목록.
- 일정 한 줄: **색 점 + 시간(시작~종료, 종료 없으면 시작만) + 제목**.
- 지난 날짜의 일정은 흐리게(회색) 표시한다. 오늘 행은 상단에 구분선으로 강조한다.
- 일정이 하나도 없으면 빈 상태 문구를 표시한다.

### R-74. 일정 색상 구분
- 이벤트 색은 출처·종류 기준으로 구분한다:
  - 숙제 마감(`hw`) = 주황(#f97316)
  - 로컬 일정(`sched`, source=local) = 파랑(#3b82f6)
  - 구글 학생 캘린더(google-student) = 보라(#8b5cf6)
  - 구글 가족 캘린더(google-family) = 초록(#10b981)
- 월간 상세 패널/바텀시트도 같은 색을 사용한다.

### R-75. 종료 시각(endAt) 데이터 추가
- `calendar_events`에 `end_at TIMESTAMPTZ`(nullable)를 추가한다. 기존 행은 NULL 유지.
- 구글 캘린더 조회 시 `end.dateTime`을 함께 가져와 내려준다.
- bootstrap 응답의 이벤트에 `endAt`(nullable)을 포함한다.
- 시드 데이터의 수업/일정형 이벤트에 적절한 종료 시각을 넣는다.

## 수용 기준 (Acceptance Criteria)
1. 뷰 토글에 월/주/일정 3개가 보이고 전환이 즉시 반영된다.
2. 주간 뷰에서 종료 시각이 있는 일정은 시간 길이에 비례한 블록으로, 없는 일정은 1시간 블록으로 그려진다.
3. 주간 뷰 오늘 열에 현재 시각 위치에 빨간 선이 보인다 (다른 열에는 없음).
4. 겹치는 두 일정이 좌우로 나뉘어 둘 다 읽을 수 있다.
5. 일정 뷰에서 지난 날짜는 흐리게, 오늘 날짜는 파란 원으로 표시된다.
6. 필터 칩을 끄면 세 뷰 모두에서 해당 종류가 사라진다.
7. 타입체크(`tsc`)와 빌드가 통과한다.

## 범위 밖
- 주간 뷰에서 드래그로 일정 생성/이동
- 종일(all-day) 이벤트 전용 상단 행
- 일정 뷰 무한 스크롤(다음 달 이어보기) — 데이터가 이번 달 범위라 이번 달만 표시
